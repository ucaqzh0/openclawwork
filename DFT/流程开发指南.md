# DFT 流程开发指南

> 所有流程（频率计算、GO、GR、后续新增）必须满足以下四要素。

---

## 四要素（必选）

| 要素 | 说明 | 实现方式 |
|------|------|----------|
| **任务拆分** | 流程可拆成独立 tool，每步可单独执行 | tool_01, tool_02, ... 结构，各 auto/*.ipynb 可独立运行 |
| **任务清单模板** | 发给 clio 或本机执行时的标准清单 | 每流程有 `任务清单_XXX.md`，从 `_template/任务清单_模板.md` 复制后填写 |
| **移交检查** | 提交/移交前必检项 | 每流程有 `shared/pre_check.py` 或 `移交检查.md`，可含自动检查脚本 |
| **双模式支持** | xiaozhua 与 clio 均可执行 | `config.py` 统一配置，路径按执行机填写；任务清单中注明执行机 |

---

## 开发步骤

### 1. 创建流程目录

```bash
mkdir -p DFT/新流程名/shared
mkdir -p DFT/新流程名/tool_01_xxx/auto
mkdir -p DFT/新流程名/tool_01_xxx/debug
```

### 2. 复制模板

- 从 `_template/任务清单_模板.md` 复制为 `新流程名/任务清单_新流程名.md`
- 从 `_template/移交检查_模板.md` 复制为 `新流程名/shared/移交检查.md`
- 填写流程特有的配置项、检查项、执行步骤

### 3. 创建 config.py

在 `新流程名/shared/config.py` 中定义该流程的配置变量（路径、分区等），并注明 xiaozhua/clio 路径差异。

### 4. 实现 pre_check（如有提交前检查）

- 编写 `shared/pre_check.py` 或 notebook
- 在 `移交检查.md` 中说明检查项与使用方式

### 5. 实现 tool

- 每个 tool 有 `auto/*.ipynb`，可独立运行
- 从 `shared/config` 读取配置
- 在 `流程映射.md` 中写明步骤与 tool 对应关系

---

## 任务清单填写要点

- **1. 任务信息**：任务名称、日期、执行机（xiaozhua/clio）
- **2. 配置项**：该流程 config.py 中需设置的变量及本机应填值
- **3. 移交检查**：该流程特有的提交前/移交前必检项
- **4. 执行步骤**：该流程的 tool 列表，可勾选
- **5. 独立执行说明**：哪些 tool 可单独运行、前置条件

---

## 双模式说明

- **xiaozhua**：在服务器上直接运行，config 填服务器路径
- **clio**：将任务清单发给执行人，执行人在 clio 上填 config（clio 可访问路径）后按清单执行
- 同一份代码，不同 config 值，即可在两种机器上运行
