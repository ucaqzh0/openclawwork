# 频率计算流程映射（5步 -> 4个tool）

1. Step1 输入与结构准备 -> `tool_01_结构准备`
2. Step2 频率任务提交 -> `tool_02_提交调度`
3. Step3 任务监控与必要重提 -> `tool_02_提交调度`
4. Step4 结果提取与汇总 -> `tool_03_结果解析`
5. Step5 虚频修正（HAS_IMAG 时触发）-> `tool_04_频率修正`

执行原则：
- 总流程看 `运行规则.md`
- 每步优先调用对应 tool 目录下的 `auto/` 文件
- 出问题优先看该 tool 的 `debug/`，再看 `shared/总调试规则.md`
- Step5 为条件分支：仅当 Step4 检查结果为 `HAS_IMAG` 时进入；修正后重提任务，回到 Step3 监控
