# GR_main_DEBUG — 调度执行补丁（按用户最新口令）

更新时间：2026-03-13

## Bugfix A（对应调度步骤2）
### 问题
资源汇报后助手直接提交任务，缺少用户最终确认。

### 修订规则（强制）
1. 先完成资源汇报（young / young.ng / 新增服务器，查不到写 `N/A`）。
2. **汇报后必须暂停**，明确等待用户二次指令：
   - 是否立即运行
   - 是否有特殊情况（参数/机器/排队策略）
3. 只有收到用户明确“开始/运行/提交”指令后，才允许执行提交。
4. 未收到确认时，禁止自动提交。

### 口令模板（执行时）
- "资源已汇报。请确认：是否现在提交？有无特殊情况需要覆盖默认调度？"
