# build / GO / 频率 — 接口规范（统一 folder 接口）

> 所有模块统一使用 build 设计的目录接口：
> `absorption/1_body|2_body/{GO,TEST,Frequency,Final}`

---

## 1. 统一目录接口（唯一标准）

```text
absorption/
  1_body/
    GO/
    TEST/
    Frequency/
    Final/
  2_body/
    GO/
    TEST/
    Frequency/
    Final/
```

说明：
- `1_body`：单吸附
- `2_body`：双吸附
- 目录名大小写按当前实现使用 `1_body/2_body`（下划线小写）

### 1.1 absorption 主目录（建议完整层级）

```text
absorption/
  1_body/{GO,TEST,Frequency,Final}
  2_body/{GO,TEST,Frequency,Final}
  GO_best/
    1_body/{GO,TEST}
    2_body/{GO,TEST}
```

- `absorption/1_body|2_body`：运行主目录（build、GO、频率都围绕这里流转）
- `absorption/GO_best`：GO 筛选后的“评级区/基准区”，供频率与后续流程溯源

### 1.2 频率计算 folder（在 absorption 语境下）

频率相关的有效结构落在：
- `absorption/1_body/Frequency`
- `absorption/2_body/Frequency`

语义：
- 有虚频但已修正的结构先进入 `Frequency/`
- 完成一次 SCF 并确认后，再写入对应 `Final/`

### 1.3 GO_best 评级（1_body / 2_body）

`GO_best` 统一按 body 分层并保留 `GO/TEST`：

```text
absorption/GO_best/
  1_body/
    GO/
    TEST/
  2_body/
    GO/
    TEST/
```

评级规则建议：
- **GO**：每个吸附物（或 case）最低能结构（生产基准）
- **TEST**：待人工确认/存在争议/策略保留结构
- 导出最小包：`CONTCAR -> .vasp` + `OUTCAR`

---

## 2. 各模块职责边界

### build（结构建立）
- 负责生成结构文件（`.vasp`）与标号清单
- 写入目标：`GO/` 或 `TEST/`
- 不做计算，不做能量筛选，不做虚频判定

### GO（计算 + best 筛选）
- 从 `GO/` 读取任务并完成计算
- 做最低能筛选，生成 GO_best（1_body 与 2_body 分开）
- 导出最小结果包：`CONTCAR -> .vasp` + `OUTCAR`
- 回写到 `absorption/GO_best/1_body|2_body/{GO,TEST}`（作为频率与最终归档输入）

### 频率计算（虚频判定 + 最终归档）
- 从 GO_best 溯源到目标结构
- 无虚频：写入对应 body 的 `Final/`
- 有虚频且修正完成：先写入 `Frequency/`（供一次 SCF），SCF 后再写入 `Final/`

---

## 3. 文件级接口（按 folder）

### `GO/`
- 输入（build -> GO）：
  - `*.vasp`（结构）
  - `*_atom_index.txt`（可选）
- 输出（GO 内部计算后）：
  - 对应 job 的计算目录与结果

### `TEST/`
- 输入（build）：
  - 新结构 `.vasp`（人工检查用）
- 输出：
  - 人工确认结果（可手动迁移到 `GO/`）

### `Frequency/`
- 输入（频率模块）：
  - 有虚频但已修正完成的结构 `.vasp`
  - 可附带 `OUTCAR`（修正阶段结果）
- 输出：
  - 一次 SCF 的结果（`CONTCAR/OUTCAR`）
  - 满足条件后迁移到 `Final/`

### `Final/`
- 输入（频率模块最终落地）：
  - `CONTCAR` 转写的 `*.vasp`
  - 对应 `OUTCAR`
- 语义：
  - 最终确认可用结构（后续 NEB/汇报从这里读取）

### `thermol/Cu_100_absorption/`
- 输入（tool_04）：
  - `1_body/Final` + `2_body/Final` 下的 `.vasp` 与 `OUTCAR`
- 输出：
  - `thermol/Cu_100_absorption/1_Body/**`
  - `thermol/Cu_100_absorption/2_Body/**`
  - `thermol/Cu_100_absorption/address_manifest.tsv`
- 语义：
  - 作为可复用索引池，避免重复计算

## 4. 命名与提取策略（默认 + 灵活）

默认策略：
- 提取范围：全部结构（all）
- body 范围：`1_body` + `2_body`
- 文件：`CONTCAR->.vasp` + `OUTCAR`

灵活策略（参数化）：
- `extract_scope = all | selected`
- `selected_species = [...]`
- `selected_cases = [...]`
- `body_scope = ['1_body', '2_body']`
- `final_policy = strict_no_imag | include_corrected_after_scf`

---

## 5. tool 对应建议

- `build/tool_01`：目录与接口定义（本规范）
- `build/tool_02`：单吸附结构生成，写 `1_body/GO`（可选 `TEST`）
- `build/tool_03`：双吸附结构生成，写 `2_body/GO`（可选 `TEST`）
- `build/tool_04`：Final 汇总与地址清单生成（不判虚频）
- `Global_optimization/tool_03`：GO_best 筛选与导出
- `频率计算`：虚频判定、修正流转、`Frequency -> Final` 落地
