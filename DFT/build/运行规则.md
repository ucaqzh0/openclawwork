# build — 运行规则

## 目标
只做“最基本结构”的建立：位点准确 + 分子结构合理。

本目录**不包含 NEB 导出**，也不生成 `ini/fin`、不涉及 `nebmake.pl`。

## 工程约定
- tool 目录分四步：`tool_01~tool_04`
- shared 提供可复用的配置加载与结构生成/对齐用工具（后续可扩展到其它阶段）
- 执行采用双模式（`xiaozhua` / `clio`）：配置路径按执行机可访问性填写

## 构建阶段的移交检查（结构层面）
- build 产物统一使用 `.vasp` 格式（例如 `job_xxx.vasp`）
- 命名与目录层级与 `@absorption` 的 `1_body/2_body` 保持一致
