# 🦞 OpenClaw Work Repository

> 自动化任务的**主文件聚合仓库**，不是任务执行仓库。

## 这个仓库是做什么的？

`openclaw_work` 用于统一存储自动化任务的核心资产，供每只 🦞 在执行时按需复制（copy）对应工作文件。

它的定位是：

- ✅ 汇聚任务主流程与调试资产
- ✅ 提供可复用的工具与模板
- ❌ 不用于独立执行某个具体任务

---

## 使用方式（核心原则）

每只 🦞 在执行任务时：

1. 从本仓库中定位对应任务类型与方法
2. copy 需要的工作文件到自己的执行环境
3. 按主流程运行，并参考 debug 文件排查问题

---

## 分支与目录约定

### 1) 每种任务类型对应一个 branch

例如（示意）：

- `task-type-a`
- `task-type-b`
- `task-type-c`

### 2) 每种任务方法对应 branch 下的一个文件夹

目录结构示意：

```text
<task-type-branch>/
└── <method-name>/
    ├── main_flow.md
    ├── debug.md
    └── tools/
        ├── xxx.ipynb
        ├── yyy.ipynb
        └── ...
```

其中：

- `main_flow.md`：🦞 完成任务的主流程说明
- `debug.md`：常见问题、排障路径、调试记录
- `tools/`：该方法依赖的已开发完成的工具主流（以 `.ipynb` 为主）

---

## 维护建议

- 新增任务类型 → 新建 branch
- 新增方法 → 在对应 branch 下新增方法文件夹
- 更新流程或排障经验 → 同步更新 `main_flow.md` / `debug.md`
- 工具成熟后再进入 `tools/`，保持可用与稳定

---

## 目标

让每只 🦞 都能在最短时间内拿到正确文件、跑通主流程、快速定位问题。
